## Short Study Description

This study is a secondary analysis of restricted-use Add Health and AHAA data. The analysis focuses on how transcript-coded course structure, transcript-coded academic evaluation, curriculum exposure, and later educational outcomes relate within the same respondents. The project does not involve contact with participants, intervention, or re-identification efforts. It uses existing restricted-use survey and transcript-linked educational records to distinguish school-pipeline mechanisms behind divergence between tested achievement surfaces and school-evaluated performance surfaces.

[INFERENCE]

---

## Why Restricted Data Are Needed

Restricted-use transcript-linked data are required because the public-use Add Health files do not include the transcript-coded course sequence, transcript-coded GPA/credits/failures, transcript timing/linking indicators, curriculum summaries, or transcript weights needed for the planned analysis. Without the AHAA transcript layer, the project cannot distinguish transcript-coded academic exposure from transcript-coded academic evaluation.

## Data Elements Requested

The request is intentionally narrow and limited to:

1. restricted-use Add Health `Core Files`
2. AHAA linking indicators
3. transcript course-structure files: `edumsov1`, `edumsov2`, `edu1`
4. one negative-control subject family: `edueng`
5. transcript curriculum summary files: `edutmsum`, `edutssum`
6. transcript graduation / exit indicators: `edugrad`
7. school-context OCR indicators: `eduocr`
8. transcript weights: `eduwgt`

## Human Subjects / Risk Language

The project is secondary analysis of existing restricted-use data. The main risks are informational rather than physical or intervention-related. No participant contact, recruitment, treatment, or deception is involved. The primary protections are restricted-use contractual controls, approved secure-access procedures, limited-user access, and analysis/reporting practices that avoid disclosure of identities or small-cell re-identification.

## Data Security Language

All restricted-use data will be accessed only by approved project personnel through the approved Add Health restricted-use access workflow and the required security plan for the named access location(s). Access will be limited to the minimum project personnel necessary. Restricted data will not be copied to unapproved locations or used outside the approved project scope.

## No-Contact / No-Intervention Language

This project uses existing survey and transcript-linked records only. There will be no direct participant contact, no recruitment, no follow-up, and no intervention.

## Scope-Control Language

The study is not designed to estimate a broad or generic “sex difference in intelligence.” The restricted request is limited to the minimum transcript-linked components required to distinguish transcript-coded course structure, transcript-coded academic evaluation, curriculum exposure, school context, and later educational attainment.
