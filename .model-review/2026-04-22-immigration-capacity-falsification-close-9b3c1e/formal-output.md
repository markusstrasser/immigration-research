## 5. My Top 5 Recommendations (different from the originals)
1. **What:** Emit a machine-readable newcomer-summary artifact with three distinct statistics: `ratio_of_medians`, `median_of_ratios`, and `share_zero_or_undefined_ratio_counties`.  
   **Why:** The current memo uses numbers that imply different estimands (`4.49%`, `0.21%`, `20.5x`). That is a live correctness bug, not stylistic nitpicking. Quantitatively, `4.49/0.21 = 21.38`, so the current prose cannot be literally true unless `20.5x` is a different statistic.  
   **How to verify:** Regenerate memo text only from the new summary artifact; unit test that any reported ratio exactly matches the declared estimand.

2. **What:** Remove the unqualified “lower-bound directional comparison” language unless a sensitivity grid agrees on direction.  
   **Why:** The timing mismatch (`2022–2023` IRS vs `2022` ACS 5-year) creates sign ambiguity in bias direction. This is the strongest remaining overclaim in the newcomer layer.  
   **How to verify:** Report the gap under at least four variants: base sample, excluding `MOE >= estimate`, `totpop >= 25k`, and alternative denominator presentation. Only restore directional language if all variants preserve the sign logic.

3. **What:** Recode threshold-null interval reporting so repeated random state splits are explicitly descriptive, or switch to group-valid inference (e.g. leave-one-state-out / leave-one-division-out).  
   **Why:** Wilson intervals over dependent splits look more rigorous than they are. This is a formal inference bug with low maintenance-cost to fix.  
   **How to verify:** Either (a) remove CI fields from JSON and memo, or (b) replace with intervals built from independent grouped units; document unit count used for inference.

4. **What:** Add a hard memo gate: no claim above `low causal confidence` if clean pre-surge windows show sign instability or nontrivial lead effects.  
   **Why:** The current memo already narrates the annual design as unresolved; the remaining inconsistency is the partial upgrade to “moderate causal confidence” for the broad stress-marker claim. A gate will stop this class of semantic drift.  
   **How to verify:** Summary JSON gets a field like `causal_gate_status = fail/pass`; memo generation refuses “moderate/high causal confidence” when gate fails.

5. **What:** Quantify threshold-location instability using concentration metrics, not prose alone.  
   **Why:** “Diffuse cutoff surface” is directionally plausible but not yet formalized. With `100` actual splits over `14` grid cells, you can compute entropy, HHI, top-cell share, and actual-minus-null deltas cheaply and reuse them elsewhere.  
   **How to verify:** Add these fields to `county_capacity_threshold_surface.csv`/summary JSON; define a rule like “stable location requires top-cell share > 0.30 and actual HHI at least 0.05 above null.”

## 6. Where I'm Likely Wrong
1. I may be **over-penalizing semantic ambiguity** because I only see excerpts, not the full output tables. If the raw newcomer parquet already labels ratio estimands and zero-denominator handling clearly, then L1/L5 are memo-layer issues, not pipeline-layer issues. [INFERENCE]

2. I may be **too cautious about the directional-bias claim**. If the intended target is specifically domestic-origin vs foreign-origin newcomer flow, and you are comfortable abstracting away timing mismatch, then “likely understated” is defensible. My objection is that timing mismatch is large enough that “likely” should probably be downgraded to “plausible.” [INFERENCE]

3. I may be **too strict on inferential calibration** for the threshold-search null. If the CI fields are used only as rough descriptive summaries and not as formal uncertainty statements, then L3 matters less. But then they should be labeled descriptive, not inferential. [INFERENCE]

4. I may be showing a **production-grade bias** by wanting machine-readable summary artifacts and explicit gates. For a personal research repo, that can be overkill. The counterargument is that this repo explicitly prioritizes maintenance burden and semantic robustness over build effort, so lightweight formalization is unusually justified here. [INFERENCE]

5. I may be **overcautious in downgrading causal language**. If there are additional designs outside the excerpted files that independently support the broader stress-marker claim, then my L4 criticism would weaken. Based on the cited artifacts shown here, though, the descriptive-to-causal jump still looks too large. [INFERENCE]
